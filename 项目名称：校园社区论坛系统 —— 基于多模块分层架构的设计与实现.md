项目名称：校园社区论坛系统 —— 基于多模块分层架构的设计与实现



------



一、项目背景
随着网络社区的发展，用户对论坛系统的需求已经不仅仅停留在简单的发帖与回复，而是希望拥有集成问答、博客、社交互动、标签分类和搜索等多种功能的现代化平台。传统论坛系统在用户体验、扩展性以及前后端分离架构方面存在不足。因此，本项目基于 Java + Spring Boot 技术栈开发，旨在打造一个功能完善、易扩展的开源社区系统，满足现代用户在内容创作、互动交流和信息管理方面的需求。
二、模块分析
项目采用分层与模块化设计，为了便于理解，这里对模块进行更直观的命名和说明：

1. 用户交互模块：负责网站前台部分，包括用户登录注册、文章浏览、问答互动、评论、点赞、消息通知等功能，是最终用户最直接使用的部分；
2. 应用启动模块：用于统一整合和启动整个系统，加载各个模块配置，保证系统能以微服务的形式平稳运行；
3. 接口服务模块：提供 RESTful API 接口，定义外部调用和前端交互的规范，并作为系统功能的对外出口；
4. 核心业务模块：包含文章、用户、问答、评论、通知等核心业务逻辑，是真正处理社区功能的核心所在；
5. 基础支撑模块：负责数据库连接、缓存、搜索引擎接入、文件存储抽象等，为核心业务提供底层支持；
6. 公共工具模块：封装日志、异常处理、工具类、统一响应格式等通用功能，为其他模块提供便利。

所涉及到的功能有：
1）账号与身份 - 注册、登录、退出（含表单校验）| 找回/重置密码（任选其一，或先留接口）| 个人资料查看与编辑（头像、昵称、简介）；
2）内容发布与浏览 - 发帖（文章）| 帖子列表：分页、按时间排序 | 详情页：正文、作者信息、发布时间；
3）互动功能 - 评论/回复（单层或简单两级）| 点赞（帖子与评论至少支持一种）| 收藏/取消收藏（可用来做“我的收藏”）；
4）分类与检索 - 标签/分类展示与筛选（至少标签）| 关键字搜索（标题/内容的简单匹配）；
5）消息与通知 - 与我相关：评论了我、回复了我、点赞了我（列表+未读数）| 站内通知阅读状态（已读/未读）；
6）个人中心 - 我发布的帖子/问题 | 我的评论、我的收藏；
7）基础管理 - 管理员登录 | 内容审核：下架/删除帖子或评论

8）私信功能，用户可以通过别人的帖子之类的，直接私信/加好友

9）分区：比如考研/保研/竞赛/二手交易等专栏

10）管理员部分：设立榜单，比如点赞/收藏/观看等最多的榜单，周榜月榜

 三、使用技术

1. 用户交互模块：

\- 前端框架：Vue.js
\- 样式库：Bootstrap
\- 富文本编辑：Markdown 编辑器（mavon-editor）
\- 页面渲染：Thymeleaf（部分服务端渲染）

2. 应用启动模块：

\- Spring Boot 用于统一配置管理和模块整合，简化部署与启动。

3. 接口服务模块：

\- Spring MVC 提供 REST API 接口
\- fastjson 用于 JSON 数据序列化与反序列化
\- Swagger（可扩展）用于接口文档生成

4. 核心业务模块：

\- MyBatis 负责 ORM 映射
\- PageHelper 插件用于分页查询
\- Spring AOP 用于业务切面处理（如日志、权限校验）

5. 基础支撑模块：

\- 数据库：MySQL 8.0
\- 连接池：HikariCP 提供高性能数据库连接
\- 缓存：Redis（可选配置）
\- 文件存储：本地磁盘存储，支持扩展至云存储（如阿里云 OSS、七牛云等）
\- 搜索引擎：预留 Elasticsearch 扩展接口

6. 公共工具模块：

\- 日志：SLF4J + Logback
\- 异常与错误码统一封装
\- 公共工具类：日期工具、字符串工具、通用响应封装
非功能性目标包含：
1）可用性：页面加载与表单反馈清晰、错误提示友好；
2）可靠性：核心流程（注册登录、发帖、评论）可回归；
3）可维护性：接口返回统一格式、错误码统一、基础日志；
4）可部署性：一键启动（本地/课堂环境），提供 README「如何运行」。
